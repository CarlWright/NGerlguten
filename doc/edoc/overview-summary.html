Authors: Joe Armstrong , Mikael Karlsson , Johan Bevemyr , Sean Hinde , Håkan Stenholm .

ErlGuten is a typesetting system for producing PDF from XML.

Erlguten is free software published under the MIT license.

The word ErlGuten is built from Erlang plus Gutenberg.

The goal of ErlGuten is to produce typographic quality output - this means that we aim at the highest quality possible for the typeset text. The ErlGuten text justification algorithms produce paragraph-optimized text with fully kerned data, normally the production of such text cannot be done by everyday word-processing programs but requires the use of dedicated typesetting programs.

== Download ==

The transformation software is written in Erlang. To run the system you will need to install Erlang, which is available from www.erlang.org.

=== Release information ===

This is release 2.5 of ErlGuten. Following release 2.1, it has been maintained for the last years at Klarna AB (formerly Kreditor). It is now published under the MIT license, with permission from all the authors. The previous release (2.1) can be found at http://www.sics.se/~joe/erlguten-2.1/erlguten.html.

=== Future releases ===

Future releases of ErlGuten will need to address the following problems:

  • Font embedding - so far we can only use the PDF embedded type 1 postscript fonts - I'd like to extend this to cover all type 1 postscript fonts, together with truetype fonts.
  • A content management system for the production of digital newspapers.
  • Plug-ins. I want to make a plug in architecture - so effects like drop-paragraphs etc. will all be implemented through a standard plug-in interface.
  • Viewers - I'd like to connect ErlGuten to the freetype project and make a GUI with the same interface as provided by the eg_pdf module.
